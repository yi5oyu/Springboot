# 인증(Authentication) / 인가(Authorization)

인증: 사용자의 신원을 검증하여 로그인 처리  
인가: 검증된 사용자가 특정 리소스에 접근할 권한이 있는지 확인

</div>

## Spring Security

1. **SecurityFilterChain**
   * 보안 관련 필터(CSRF, CORS, 인증, 인가 등 처리)
   * `SecurityConfig.java`에서 설정한 내용이 이 필터 체인에 반영

2. **Authentication(인증 객체)**
   * 로그인한 사용자의 정보(ID, PW, 권한 등)를 담는 객체
   * `Principal`: 사용자 식별 아이디(User 객체)
   * `Credentials`: 비밀번호(주로 인증 후 삭제됨)
   * `Authorities`: 사용자의 권한 목록(ROLE_USER, ROLE_ADMIN 등)

3. **SecurityContextHolder**
   * 보안 컨텍스트(SecurityContext)를 저장하는 저장소
   * `ThreadLocal`을 사용해 **같은 스레드 내에서는 어디서든 인증 정보에 접근**해 Controller에서 `@AuthenticationPrincipal`로 사용자 정보를 가져올 수 있음

## 인증 방식 비교

### 세션/쿠키(Session/Cookie) - Stateful
- 서버가 사용자의 로그인 상태(세션)를 메모리나 DB에 저장하고 클라이언트에게는 세션 ID(JSESSIONID)를 쿠키로 발급

### JWT(JSON Web Token) - Stateless
- 로그인 성공 시 서버가 사용자 정보를 암호화한 **토큰(Token)**을 발급, 서버는 별도의 세션을 저장하지 않음
