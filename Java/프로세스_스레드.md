# 프로세스/스레드

프로그램 실행 / 스케슐링

</div>

## 프로세스(Process)

`운영체제로부터 자원을 할당받아 실행 중인 프로그램의 단위`

```
독립적인 메모리 공간(Code, Data, Heap, Stack)을 가짐
프로세스 간에는 메모리를 공유하지 않음(IPC: Inter-Process Communication 필요)
```

### 멀티 프로세스

`하나의 응용 프로그램을 여러 개의 프로세스로 구성해 각 프로세스가 하나의 작업을 처리`

```
프로세스 하나가 비정상 종료되어도 다른 프로세스에 영향을 주지 않음
Context Switching 비용 큼: CPU 캐시 메모리 초기화 등 무거운 작업 필요
통신 복잡: IPC 사용으로 코드가 복잡하고 통신 비용 발생
```

## 스레드

`프로세스 내에서 실행되는 흐름의 단위`

```
프로세스가 할당받은 자원(Code, Data, Heap)을 공유
스레드마다 독립적인 스택(Stack)과 PC 레지스터(PC Register)를 가짐
```

### 멀티 스레드

`하나의 응용 프로그램을 여러 개의 스레드로 구성해 각 스레드가 작업을 처리`

```
빠른 Context Switching: 캐시 메모리를 비울 필요가 없어 전환 속도가 빠름
하나의 스레드에서 예외가 발생하면 프로세스 전체가 종료될 수 있음
여러 스레드가 공유 자원에 동시 접근할 때 데이터 불일치 발생 가능
```

> Context Switching
> CPU가 현재 작업 중인 프로세스/스레드를 중단하고 다른 프로세스/스레드로 전환하는 과정
